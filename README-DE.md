## ⚡ Überblick

**MiroFish** ist eine KI-Vorhersage-Engine der nächsten Generation, angetrieben von Multi-Agenten-Technologie. Durch die Extraktion von Realitäts-Saaten aus der echten Welt (etwa aktuelle Nachrichten, Politikentwürfe oder Finanzsignale) baust du automatisch eine hochauflösende Parallelwelt auf. In diesem Raum interagieren tausende intelligente Agenten mit eigener Persönlichkeit, Langzeitgedächtnis und Verhaltenslogik frei miteinander und durchlaufen eine soziale Entwicklung. Aus der Vogelperspektive kannst du dynamisch Variablen einspielen, um zukünftige Verläufe präzise abzuleiten – **probe die Zukunft im digitalen Sandkasten und lass deine Entscheidungen nach unzähligen Simulationen gewinnen**.

> Du musst nur: Ausgangsmaterial hochladen (Datenanalyseberichte oder spannende Romangeschichten) und deine Vorhersageanforderung in natürlicher Sprache beschreiben</br>
> MiroFish liefert dir zurück: einen detaillierten Vorhersagebericht und eine hochauflösende digitale Welt zum tiefen Interagieren

### Unsere Vision

MiroFish widmet sich dem Aufbau eines Schwarmintelligenz-Spiegels, der die Realität abbildet. Indem wir das kollektive Emergenzverhalten einfangen, das durch individuelle Interaktionen ausgelöst wird, durchbrechen wir die Grenzen klassischer Vorhersagen:

- **Auf der Makroebene**: Wir sind ein Proberaum für Entscheidungsträger, in dem sich Politik und Öffentlichkeitsarbeit risikofrei testen lassen
- **Auf der Mikroebene**: Wir sind ein kreativer Sandkasten für einzelne Nutzer – ob du Romanenden ableitest oder fantasievolle Szenarien erkundest, alles kann unterhaltsam, verspielt und leicht zugänglich sein

Von ernsthaften Vorhersagen bis zu spielerischen Simulationen lassen wir jedes „Was wäre wenn" sein Ergebnis sehen und machen es möglich, alles vorherzusagen.

## 🔄 Workflow

1. **Graph-Aufbau**: Extraktion der Saat & Injektion von individuellem/kollektivem Gedächtnis & GraphRAG-Aufbau
2. **Umgebungs-Setup**: Entitätsbeziehungs-Extraktion & Persona-Generierung & Injektion der Agenten-Konfiguration
3. **Simulation**: Parallele Zwei-Plattform-Simulation & automatisches Parsen der Vorhersageanforderung & dynamische Aktualisierung des zeitlichen Gedächtnisses
4. **Berichterstellung**: ReportAgent mit umfangreichem Toolset für die tiefe Interaktion mit der Umgebung nach der Simulation
5. **Tiefeninteraktion**: Chatte mit jedem beliebigen Agenten in der simulierten Welt & interagiere mit dem ReportAgent

## 🚀 Schnellstart

### Option 1: Bereitstellung aus dem Quellcode (empfohlen)

#### Voraussetzungen

| Tool | Version | Beschreibung | Installation prüfen |
|------|---------|-------------|-------------------|
| **Node.js** | 18+ | Frontend-Laufzeitumgebung, inklusive npm | `node -v` |
| **Python** | ≥3.11, ≤3.12 | Backend-Laufzeitumgebung | `python --version` |
| **uv** | Aktuellste Version | Python-Paketmanager | `uv --version` |

#### 1. Umgebungsvariablen konfigurieren

```bash
# Beispiel-Konfigurationsdatei kopieren
cp .env.example .env

# Die .env-Datei bearbeiten und die benötigten API-Schlüssel eintragen
```

**Erforderliche Umgebungsvariablen:**

```env
# LLM-API-Konfiguration (unterstützt jede LLM-API im OpenAI-SDK-Format)
# Empfohlen: Alibaba Qwen-plus-Modell über die Bailian-Plattform: https://bailian.console.aliyun.com/
# Hoher Verbrauch, versuche zuerst Simulationen mit weniger als 40 Runden
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Zep Cloud Konfiguration
# Das monatliche Gratiskontingent reicht für einfache Nutzung aus: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. Abhängigkeiten installieren

```bash
# Alle Abhängigkeiten mit einem Klick installieren (Root + Frontend + Backend)
npm run setup:all
```

Oder Schritt für Schritt installieren:

```bash
# Node-Abhängigkeiten installieren (Root + Frontend)
npm run setup

# Python-Abhängigkeiten installieren (Backend, erstellt automatisch eine virtuelle Umgebung)
npm run setup:backend
```

#### 3. Dienste starten

```bash
# Frontend und Backend gemeinsam starten (im Projekt-Root ausführen)
npm run dev
```

**Dienst-URLs:**
- Frontend: `http://localhost:3000`
- Backend-API: `http://localhost:5001`

**Einzeln starten:**

```bash
npm run backend   # Nur Backend starten
npm run frontend  # Nur Frontend starten
```

### Option 2: Docker-Bereitstellung

```bash
# 1. Umgebungsvariablen konfigurieren (wie bei der Quellcode-Bereitstellung)
cp .env.example .env

# 2. Image pullen und starten
docker compose up -d
```

Liest standardmäßig die `.env` aus dem Root-Verzeichnis und mappt die Ports `3000 (Frontend) / 5001 (Backend)`

> Eine schneller erreichbare Mirror-Adresse ist als Kommentar in `docker-compose.yml` hinterlegt, bei Bedarf ersetzen.
